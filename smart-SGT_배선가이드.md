# smart-SGT 배선 가이드 (Arduino Uno R4 WiFi)

부품별로 **선 하나하나 어디에 꽂는지** 정리했습니다.

## 0. 이 시스템이 하는 일

| 상황 | 감지 방법 | 자동 대응 |
|---|---|---|
| 🔥 **화재** | HuskyLens#1 불꽃(색) **AND** MQ-2 가스 ≥400 이 3초 안에 함께 잡힘 | 부저 경보 · 내장 매트릭스 전체 점멸 · 환기 모터 · 창문 닫힘 · 대시보드 긴급 · 음성 "대피하세요" · 텔레그램 · **주변 소방서 자동 안내** |
| 🚨 **낙상** | HuskyLens#2가 쓰러짐(ID2)을 **1초 이상** 연속 인식 | 대시보드 알림 · 음성 안내 · 스피커 방송 · 텔레그램 · **주변 응급의료기관 자동 안내** (3분 미응답 시 119 긴급 안내) |
| 🆘 **도움 요청** | TFT HELP 터치 또는 대시보드 버튼 | 대시보드 접수함 · 텔레그램 |
| 🚪 **출입 인원** | 초음파 2개의 감지 순서로 방향 판단 | 실내 인원 ±1 · TFT/대시보드 표시 · 시간대별 CSV 기록 |
| 💡 **교실 전등** | 실내 인원 1명 이상 | 흰색 LED ON, 0명이면 OFF |
| 🌡️ **교실 환경** | DHT11 온·습도, MQ-2 가스 | 대시보드 카드 · TFT 표시 |
| ⏰ **시간표·이동수업** | 대시보드 시간표 + 실내 인원 | 깨우기 방송 · 이동수업 미이동 안내 방송 |


> ⚠️ 공통 규칙 3가지
> 1. **모터·서보는 반드시 별도 5V 전원**에서 받고, **GND만 보드와 공통**으로 묶습니다. (보드 5V 직결 = 과열·연기 원인)
> 2. R4에서 **A4 = SDA, A5 = SCL** 입니다. I2C 전용이라 다른 부품에 쓰면 안 됩니다.
> 3. 배선을 바꿀 땐 **항상 전원을 뽑고** 작업합니다.

## 2. HuskyLens ×2 — T핀·R핀 어디에?

HuskyLens 4핀 커넥터는 **왼쪽부터 T · R · − · +** 순서입니다.
케이블 색: **초록 = T, 파랑 = R, 검정 = −, 빨강 = +**

### HuskyLens #1 (화재 · 색 인식) — I2C 모드
I2C 모드에서는 **T가 SDA, R이 SCL** 역할을 합니다.

| HuskyLens#1 | → | 아두이노 |
|---|---|---|
| **T** (초록) | → | **A4 (SDA)** |
| **R** (파랑) | → | **A5 (SCL)** |
| **−** (검정) | → | GND |
| **+** (빨강) | → | 5V |

본체 설정: `General Settings → Protocol Type → **I2C**`

### HuskyLens #2 (낙상 · 사람) — UART(Serial1) 모드
UART 모드에서는 **T = TX(보내는 선), R = RX(받는 선)** 입니다.
통신선은 **서로 엇갈리게(크로스)** 꽂아야 합니다.

| HuskyLens#2 | → | 아두이노 | 이유 |
|---|---|---|---|
| **T** (초록, TX) | → | **D0 (RX)** | 허스키가 보내는 걸 보드가 받음 |
| **R** (파랑, RX) | → | **D1 (TX)** | 보드가 보내는 걸 허스키가 받음 |
| **−** (검정) | → | GND | |
| **+** (빨강) | → | 5V | |

**T→D0, R→D1** 입니다. 반대로 꽂으면 아무 값도 안 들어옵니다 (`#DBG`에 `HL2=NO`).
본체 설정: `Protocol Type → **Serial**`, 통신속도 9600.

> HuskyLens는 전류를 많이 먹습니다(최대 320mA). 화면이 자꾸 꺼지면 **HuskyLens의 USB 단자로 별도 전원**을 주세요. 4핀과 USB를 동시에 꽂아도 됩니다.

## 3. DC 모터 (환기 팬) — 드라이버 종류별

**어떤 방식이든 모터 전원은 보드가 아니라 별도 5V에서 받습니다.**

### 방법 A — L9110S 모듈 (2핀 입력, 가장 간단)

| L9110S | → | 연결 |
|---|---|---|
| **INA (또는 IA)** | → | **D8** |
| **INB (또는 IB)** | → | **GND** (한 방향만 쓰면 GND 고정) |
| **VCC** | → | 별도 5V (+) |
| **GND** | → | 별도 5V (−) **그리고 아두이노 GND** |
| **MOTOR A / B** | → | 모터 두 선 |

### 방법 B — L298N 모듈

| L298N | → | 연결 |
|---|---|---|
| **IN1** | → | **D8** |
| **IN2** | → | **GND** |
| **ENA** | → | 점퍼 캡 꽂아두기 (항상 활성) |
| **+12V** | → | 별도 5V (+) |
| **GND** | → | 별도 5V (−) **그리고 아두이노 GND** |
| **OUT1 / OUT2** | → | 모터 두 선 |
| 5V 핀 | → | 사용 안 함 (점퍼 있으면 그대로) |

### 방법 C — 트랜지스터(2N2222 / TIP120) 직접 구성

```
D8 ──[1kΩ]── 트랜지스터 B(베이스)
트랜지스터 E(이미터) ── GND (아두이노 + 별도전원 공통)
트랜지스터 C(컬렉터) ── 모터 (−)
모터 (+) ── 별도 5V (+)
모터 양단에 다이오드 1N4007 (줄무늬 쪽이 + 방향)
```

**확인 방법**: 업로드하면 부팅 자가진단에서 **모터가 1초 회전**합니다.
안 돌면 시리얼 모니터에 `TEST_MOTOR` 를 입력해 다시 시험하세요.
→ 그래도 안 돌면 ① 별도 전원 연결 ② **GND 공통** ③ IN 핀 위치 순서로 확인.

## 4. 흰색 LED (교실 전등)

LED는 **다리 길이로 방향**이 정해집니다. 반대로 꽂으면 절대 안 켜집니다.

```
A3 ──[220Ω 저항]── LED 긴 다리(+, 애노드)
                    LED 짧은 다리(−, 캐소드) ── GND
```

| 구분 | 표시 |
|---|---|
| 긴 다리 | + (애노드) → 저항 거쳐 **A3** |
| 짧은 다리 | − (캐소드) → **GND** |
| 몸통이 평평하게 깎인 쪽 | 짧은 다리(−) 쪽 |

저항은 **220Ω** (빨강-빨강-갈색). 없으면 330Ω도 됩니다.
**확인**: 부팅 자가진단에서 3번 깜빡입니다. 시리얼에 `TEST_LED` 입력해도 됩니다.
동작 조건: **실내 인원이 1명 이상이면 ON**, 0명이면 OFF.

## 5. TFT (3.2" SPI ILI9341 + 터치) — 백화 해결 배선

> ⚠️ **먼저 확인**: 3.2"는 **SPI 방식**과 **병렬(parallel) 방식** 두 종류가 있습니다.
> 핀에 `SDI(MOSI) · SCK · DC · CS · RESET` 이 있으면 **SPI**(이 가이드 그대로 사용).
> 핀에 `DB0~DB15` 또는 `LCD_D0~D7 · RD · WR · RS` 가 있으면 **병렬 방식**이라 지금 코드로는 절대 안 켜집니다.

### 5-1. 지금 꽂아야 할 선 (터치는 잠시 뺌)

| TFT 핀 | → | 아두이노 | 비고 |
|---|---|---|---|
| **VCC** | → | **5V** | 보드에 3.3V 레귤레이터 있음 |
| **GND** | → | **GND** | |
| **CS** | → | **A1** | 화면 선택 |
| **RESET** | → | **A0** | ★ 보드 RESET핀 아님! |
| **DC** (또는 RS) | → | **D10** | |
| **SDI (MOSI)** | → | **D11** | |
| **SCK** | → | **D13** | |
| **SDO (MISO)** | → | **D12** | 진단값 읽기용 |
| **LED** | → | **5V** (또는 3.3V) | 백라이트 |

### 5-2. 절대 꽂으면 안 되는 핀
- **SD_SCK, SD_MISO, SD_MOSI, SD_CS** → SPI가 충돌해서 화면이 하얘집니다. 전부 비워두세요.
- **T_IRQ, T_CLK, T_CS, T_DIN, T_DO** (터치) → 화면부터 살릴 때는 **전부 뽑아두세요.**

### 5-3. 하얀 화면이 계속되면 (순서대로)
1. **RESET을 A0로** 옮겼는지 확인 — 가장 흔한 원인
2. `tft_test.ino` 업로드 → **시리얼 모니터 115200** 열고 진단값 5줄 확인
   - 전부 `0x0` / `0xFF` → 통신 자체가 안 됨 → 4번으로
   - 값이 섞여 나옴 → 통신은 됨 → 3번으로
3. `SPI_HZ` 를 `4000000` → `2000000` 으로 더 낮춰보기
4. **5V → 3.3V 보호 저항**: 이 모듈은 3.3V용, R4는 5V입니다.
   **SCK · SDI · CS · DC · RESET 5개 선 각각에 1kΩ을 직렬로** 넣으세요.
   (아두이노 핀 ──[1kΩ]── TFT 핀)
5. 그래도 안 되면 케이블·브레드보드 접촉 불량 → 짧은 점퍼선으로 직결

### 5-4. 화면이 살아난 뒤 터치 되살리기
스케치 위쪽 `#define USE_TOUCH 0` → **1** 로 바꾸고,
TFT RESET을 다시 **보드 RESET핀**으로 옮긴 뒤 터치를 이렇게 꽂습니다.

| 터치 핀 | → | 아두이노 |
|---|---|---|
| T_CS | → | A0 |
| T_DIN | → | D11 |
| T_DO | → | D12 |
| T_CLK | → | D13 |
| T_IRQ | → | 연결 안 함 |

그다음 `CAL_MODE 1` 로 네 귀퉁이를 눌러 좌표를 확인하고 `mapTouch()` 숫자를 맞춘 뒤 `CAL_MODE 0` 으로 되돌립니다.

## 6. 나머지 센서

**초음파 HC-SR04 ×2** (등 맞대고 반대 방향 배치)

| | US1 (바깥/복도) | US2 (안/교실) |
|---|---|---|
| VCC | 5V | 5V |
| Trig | **D2** | **D4** |
| Echo | **D3** | **D5** |
| GND | GND | GND |

**DHT11** — VCC→5V, DATA→**D6**, GND→GND (3핀 모듈 기준)
**MQ-2** — VCC→5V, GND→GND, **AOUT→A2** (DOUT은 안 씀)
**능동 부저** — (+)→**D7**, (−)→GND
**서보 SG90** — 주황(신호)→**D9**, 빨강→별도 5V, 갈색→GND(공통)
**방송용 아두이노(스피커)** — SDA→**A4**, SCL→**A5**, GND 공통 (I2C 주소 8)

## 7. 켜고 확인하는 순서

1. 전원 넣기 → **부팅 자가진단**: TFT 빨강·초록·파랑 → LED 3번 깜빡 → 부저 삐 → 모터 1초 회전
2. 시리얼 모니터 115200 열기 → `#US 기준거리 …` 줄 확인 (문간을 비운 상태에서 켤 것)
3. 2초마다 나오는 `#DBG` 줄 확인

```
#DBG HL1(fire)=OK flame=0 gasFire=0 lv=0 | HL2(people)=OK ppl=1 fallRaw=0 fallHold=0ms fall=0 | US A=180cm B=176cm inside=0 gas=210 temp=24 light=0 fan=0
```

| 항목 | 뜻 | 이상하면 |
|---|---|---|
| `HL1=NO` | 화재캠 응답 없음 | A4/A5 배선, 본체 Protocol=I2C 확인 |
| `HL2=NO` | 사람캠 응답 없음 | **T→D0, R→D1** 확인, Protocol=Serial |
| `US A/B`가 항상 999 | 초음파 응답 없음 | Trig/Echo 뒤바뀜 확인 |
| `temp=0` | DHT11 미인식 | D6 배선 확인 |
| `flame`이 항상 1 | 비슷한 색을 불로 오인 | HuskyLens#1 색 다시 학습 |
